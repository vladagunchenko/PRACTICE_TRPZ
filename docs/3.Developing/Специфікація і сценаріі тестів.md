## **1. ВХІДНІ СПЕЦИФІКАЦІЇ**

Розділ «Вхідні специфікації» регламентує **входи**, необхідні для проходження всіх тестів для даного рішення. У числі таких входів можуть бути дані, введені з клавіатури, натиснення кнопок миші у відповідних місцях сторінки, датчики, комп'ютерне та інше обладнання, різні дії користувачів. 

Опис специфікацій поряд з найменуванням входу містить його *характеристики*, такі як значення (що введено), структури даних (якого типу), інтерфейси (яким чином), протоколи, бази даних, файли тощо.

Для тестування застосунку використовуються такі вхідні специфікації:

| № | Найменування входу | Характеристика | Тип даних | Спосіб введення |
|---|---------------------|----------------|-----------|-----------------|
| 1 | Логін користувача | Ім’я користувача або електронна пошта для входу в систему | Рядок (String) | Введення з клавіатури у форму авторизації |
| 2 | Пароль користувача | Пароль для входу в обліковий запис | Рядок (String) | Введення з клавіатури у форму авторизації |
| 3 | Назва завдання | Коротка назва персонального завдання | Рядок (String) | Введення у текстове поле |
| 4 | Опис завдання | Детальний опис завдання | Текст (Text) | Введення у текстову область |
| 5 | Дата виконання | Запланована дата завершення завдання | Дата (Date) | Вибір у календарі або введення у поле |
| 6 | Пріоритет завдання | Рівень важливості завдання: низький, середній, високий | Перелік значень (Enum/String) | Вибір зі списку |
| 7 | Статус завдання | Поточний стан завдання: нове, в процесі, виконане | Перелік значень (Enum/String) | Вибір зі списку |
| 8 | Пошуковий запит | Текст для пошуку завдань за назвою або описом | Рядок (String) | Введення у поле пошуку |
| 9 | Фільтр завдань | Параметри фільтрації за статусом, пріоритетом або датою | Набір значень | Вибір елементів інтерфейсу |
| 10 | Команда створення | Дія користувача для додавання нового завдання | Подія інтерфейсу | Натискання кнопки «Додати» / «Створити» |
| 11 | Команда редагування | Дія користувача для зміни існуючого завдання | Подія інтерфейсу | Натискання кнопки «Редагувати» |
| 12 | Команда видалення | Дія користувача для видалення завдання | Подія інтерфейсу | Натискання кнопки «Видалити» |
| 13 | Команда збереження | Підтвердження змін після створення або редагування | Подія інтерфейсу | Натискання кнопки «Зберегти» |
| 14 | Команда скасування | Відміна поточної дії без збереження змін | Подія інтерфейсу | Натискання кнопки «Скасувати» |
| 15 | Вибір завдання зі списку | Відкриття конкретного завдання для перегляду або редагування | Ідентифікатор / подія | Натискання на запис у списку |
| 16 | Дані бази даних | Записи про завдання, користувачів, статуси та пріоритети | JSON / таблиці БД | Автоматичне зчитування системою |
| 17 | Дані сесії користувача | Інформація про авторизованого користувача | Токен / службові дані | Формується автоматично після входу |
| 18 | HTTP-запити клієнта | Запити на створення, зміну, видалення та отримання завдань | JSON / HTTP Request | Передаються через веб-інтерфейс |

## **2. ВИХІДНІ СПЕЦИФІКАЦІЇ**

Розділ «Вихідні специфікації» регламентує **виходи**, очікувані як результат проходження тестів для вашого рішення. Виходи можуть містити дані, інформацію про час виконання операцій, повідомлення, логи, дампи пам'яті, файли, знімки екрану тощо. 

Для тестування застосунку використовуються такі вихідні специфікації:

| № | Найменування виходу | Характеристика | Тип даних | Спосіб отримання |
|---|----------------------|----------------|-----------|------------------|
| 1 | Повідомлення про успішний вхід | Підтвердження успішної авторизації користувача в системі | Рядок (String) | Відображення в інтерфейсі |
| 2 | Повідомлення про помилку входу | Повідомлення про неправильний логін або пароль | Рядок (String) | Відображення в інтерфейсі |
| 3 | Список завдань | Перелік усіх завдань користувача із основними даними | Масив об’єктів / таблиця | Відображення на сторінці |
| 4 | Відображення нового завдання | Поява нового запису після успішного створення | Об’єкт / елемент списку | Оновлення інтерфейсу |
| 5 | Оновлене завдання | Відображення змінених даних після редагування | Об’єкт / елемент списку | Оновлення інтерфейсу |
| 6 | Видалення завдання зі списку | Зникнення запису після підтвердження видалення | Зміна стану списку | Оновлення інтерфейсу |
| 7 | Повідомлення про успішне збереження | Інформування про успішне створення або зміну завдання | Рядок (String) | Спливаюче повідомлення / текст в інтерфейсі |
| 8 | Повідомлення про помилку валідації | Інформація про некоректно заповнені або порожні поля | Рядок (String) | Відображення біля полів або у формі |
| 9 | Результати пошуку | Відфільтрований список завдань відповідно до пошукового запиту | Масив об’єктів / список | Відображення на сторінці |
| 10 | Результати фільтрації | Список завдань, що відповідає обраним параметрам | Масив об’єктів / список | Відображення на сторінці |
| 11 | Зміна статусу завдання | Оновлений стан завдання після вибору нового статусу | Рядок / об’єкт | Оновлення інтерфейсу |
| 12 | Відповідь сервера | Результат обробки HTTP-запиту на клієнтські дії | JSON / HTTP Response | Отримання у клієнтській частині |
| 13 | Код статусу HTTP | Код результату виконання запиту (200, 201, 400, 401, 404, 500 тощо) | Число (Integer) | Отримання у відповіді сервера |
| 14 | Дані про помилку сервера | Повідомлення про помилки під час обробки запиту | JSON / текст | Відображення в інтерфейсі або логах |
| 15 | Логи виконання | Технічна інформація про запити, помилки, події системи | Текст / журнал | Перегляд у консолі або файлі логів |
| 16 | Час відповіді системи | Тривалість виконання операції або завантаження сторінки | Число / час | Вимірювання інструментами тестування |
| 17 | Візуальні зміни інтерфейсу | Оновлення сторінки, кнопок, повідомлень, полів після дій користувача | Стан UI | Спостереження в браузері |
| 18 | Збережені дані в БД | Фактичне додавання, оновлення або видалення записів у базі даних | Записи БД | Перевірка через сервер або інструменти БД |

## **3. СЕРЕДОВИЩЕ ТЕСТУВАННЯ**

У цьому розділі описуються *характеристики і конфігурація середовища* тестування. Такий опис може включати апаратуру, програмне оточення, інструменти, спеціалізоване програмне забезпечення тощо.

## **3. СЕРЕДОВИЩЕ ТЕСТУВАННЯ**

У цьому розділі описуються характеристики та конфігурація середовища, необхідного для проведення тестування веб-застосунку **Task Manager**. Середовище тестування формується з урахуванням обраних методів тестування, а саме: модульного, функціонального, тестування інтерфейсу, тестування зручності використання, сумісності, продуктивності та регресійного тестування.

Оскільки застосунок повинен коректно працювати як на **комп’ютерах і ноутбуках**, так і на **мобільних пристроях**, тестове середовище повинно охоплювати різні типи пристроїв, операційних систем, браузерів та інструментів перевірки.

### **Апаратне середовище**

Для проведення тестування використовуються такі пристрої:

- персональний комп’ютер;
- ноутбук;
- смартфон;
- за можливості планшет.

Мінімальні вимоги до комп’ютерних пристроїв:

- **процесор:** Intel Core i3 / AMD Ryzen 3 або вище;
- **оперативна пам’ять:** не менше 4 ГБ;
- **вільне місце на диску:** не менше 1 ГБ;
- **роздільна здатність екрана:** не менше 1366×768;
- **пристрої введення:** клавіатура, миша або тачпад;
- **мережа:** доступ до Інтернету.

Мінімальні вимоги до мобільних пристроїв:

- **операційна система:** Android або iOS;
- **наявність сучасного браузера** з підтримкою HTML5, CSS3 та JavaScript;
- **сенсорний екран** для перевірки коректності touch-взаємодії;
- **підключення до Інтернету** через Wi-Fi або мобільну мережу.

### **Програмне середовище**

Для виконання тестів використовується таке програмне забезпечення:

| № | Найменування | Призначення |
|---|--------------|-------------|
| 1 | Windows 10 / Windows 11 | Основне середовище тестування на ПК та ноутбуках |
| 2 | Android | Тестування мобільної версії застосунку |
| 3 | iOS | Тестування мобільної версії застосунку |

| 5 | Microsoft Edge | Перевірка сумісності та коректності відображення |
| 6 | Google Chrome for Android | Перевірка роботи сайту на Android-пристроях |
| 7 | Safari | Перевірка роботи сайту на iPhone та iPad |

### **Середовище відповідно до методів тестування**

Для кожного з обраних методів тестування використовується відповідне середовище та інструменти:

- **Модульне тестування** виконується у середовищі розробки для перевірки окремих компонентів системи.
- **Функціональне тестування** проводиться у браузері шляхом перевірки основних сценаріїв роботи застосунку.
- **Тестування інтерфейсу (UI testing)** виконується на різних розмірах екранів для перевірки правильності відображення елементів сторінки.
- **Тестування зручності використання (Usability testing)** проводиться на комп’ютерах і мобільних пристроях для оцінки зручності взаємодії з сайтом.
- **Тестування сумісності (Compatibility testing)** виконується за допомогою різних браузерів та сервісу BrowserStack.
- **Тестування продуктивності (Performance testing)** виконується за допомогою Google Lighthouse.
- **Регресійне тестування (Regression testing)** проводиться після внесення змін до функціоналу із використанням повторних перевірок та автоматизованих інструментів.

---

### **Налаштування тестового середовища**

Перед початком тестування необхідно виконати такі дії:

- розгорнути застосунок на локальному сервері або тестовому середовищі;
- перевірити доступність серверної частини та бази даних;
- створити тестові облікові записи користувачів;
- підготувати тестові дані для роботи із завданнями;
- перевірити коректну роботу сайту на різних роздільних здатностях екрана;
- виконати перевірку адаптивності через інструменти браузера;
- за можливості провести перевірку на реальних мобільних пристроях;
- очистити кеш браузера перед повторними тестами.

---

### **Умови проведення тестування**

Під час перевірки необхідно враховувати:

- коректну роботу сайту на комп’ютерах, ноутбуках і смартфонах;
- правильне відображення сторінок у різних браузерах;
- адаптацію інтерфейсу під різні розміри екрана;
- зручність використання елементів керування мишею та сенсорним введенням;
- правильність виконання операцій створення, редагування, пошуку та видалення завдань;
- стабільність роботи після змін у функціональності;
- швидкість завантаження сторінок і час реакції системи.
